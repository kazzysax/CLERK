Put clerk.io on any website

One script tag. A floating bubble for your customers. Clerk stays in standby by default — drafting and learning every time a human resolves a case, never guessing when it’s unsure.

How it works

  • Dangling sign — floating clerk bubble on your site
  • Customer care unit — chat panel that talks to clerk.io backend
  • Standby mode — Clerk always drafts; auto-replies only in live mode when confident
  • Always learning — every human resolution becomes training (exemplars + tone)

1. Provision embed key

Uses your admin token (same as Render ADMIN_TOKEN). Creates a merchant row + public widget key.

3. Teach clerk when humans resolve

From your helpdesk or desk UI, call:

POST /api/widget/learn
Authorization: Bearer ADMIN_TOKEN
{
  "merchantId": "…",
  "customerMessage": "…",
  "humanReply": "…",
  "ticketHash": "optional",
  "sessionId": "optional"
}

Or signed webhook POST /webhooks/resolved-by-human with humanReply — Clerk always learns.

4. SQL (once)

In Supabase SQL editor run supabase/schema-widget.sql (after the core schemas).
